# src/render/assets/ — asset loading & manifest

Loads the real CC0 GLBs / HDRIs / image textures the renderer bakes into scenes.

- **`loader.ts`** — `loadGltf` (meshopt-decoded), `loadHdr` (RGBE/IBL),
  `loadTexture`, each with a promise cache so every consumer shares one
  parse per URL. Results are treated as immutable — **clone before mutating**;
  `releaseGltf(url)` drops the cache entry after geometry is extracted.
  Loads run through per-type concurrency queues (smaller on constrained mobile).
- **`preload.ts`** — boot gate. Subsystems call `registerPreload(promise)` at
  import time; `startGame` awaits `assetsReady()` so `build*()` can read resolved
  assets synchronously. New module-load fetch → register it here.
- **`media.ts`** — `assetUrl(logical)`: passthrough in dev, content-hashed lookup
  via the manifest in prod.
- **`stats.ts`** — in-memory load timing. `loader.ts`/`preload.ts` feed it via
  `assetLoadStarted`/`recordAssetLoad`/`recordPreloadWait`; `assetTimingSnapshot()`
  rolls samples up by type for diagnostics. Dev instrumentation only — no UI text.
- **`manifest.generated.ts`** — **GENERATED by
  `scripts/build_media_manifest.mjs`. YOU MUST NOT hand-edit it.** It maps logical
  paths → hashed `/media/...` URLs. Regenerate via `npm run build`.

Actual asset files live under **`public/`** (`models/`, `env/`, `textures/`,
`vfx/`) — add files there; loaders fetch by their public-relative path.
